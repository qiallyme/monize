# v1.11.1

Hot on the heels of the big internationalization release, this update **more than doubles the languages Monize speaks**, teaches the AI Assistant to **make changes for you (with your approval)**, and rounds out the new Monthly Breakdown report and payee-categorization tools. It also adds a **net-worth composition chart**, **email verification** for new sign-ups, and a sweeping pass to make per-row actions and navigation feel the same everywhere.

## Languages

### Ten more languages, plus regional English

Monize now speaks **nineteen languages**. On top of the nine from v1.11.0, this release adds full translations for **Simplified Chinese (简体中文), Traditional Chinese (繁體中文), Russian (Русский), Japanese (日本語), Korean (한국어), Hindi (हिन्दी), Turkish (Türkçe), Ukrainian (Українська), Indonesian (Bahasa Indonesia), and Vietnamese (Tiếng Việt)**. Every screen, email, and error message is translated, using established financial terminology so the numbers read naturally in each language.

### Pick the English that matches your region

Monize now offers **American (en-US), Canadian (en-CA), and British (en-GB) English**, so you can have "Checking" instead of "Chequing", "Color" instead of "Colour", or "Favorite" instead of "Favourite" — whichever feels like home. The regional variants only change the words that actually differ and inherit everything else, so they stay complete as Monize grows. There's also a new **"Match my browser"** language option that follows your browser's regional settings, and the date- and number-format pickers now show a **live sample** of how each option will look.

## AI Assistant

### The Assistant can now make changes — only when you approve

The in-app AI Assistant can now **propose changes**, not just answer questions. Ask it to add a transaction, categorize one, or create a payee, and it shows you a **confirmation card** with a full preview of exactly what it will do. **Nothing is saved until you press Approve.** The model itself never holds the keys — every proposed write is signed, re-validated against your account, and re-checks that the record belongs to you before it's committed, with a daily write limit as a backstop.

### Smarter payee handling when adding transactions

When you (or the Assistant) create a transaction with a payee name, Monize now **matches it to an existing payee** — by exact name, alias, or fuzzy, punctuation-insensitive matching — links the transaction to it, and inherits that payee's default category. If there's no match, it **creates the payee for you** by default (you can opt out). The same logic is shared by the AI Assistant and the MCP server, so both behave identically, and abbreviated or lightly misspelled payee names now resolve correctly.

## Connect Monize to AI tools (MCP)

### A sturdier, better-documented MCP server

The Model Context Protocol server — which lets external AI tools query your finances — gained **output schemas and annotations for all of its tools**, so connecting clients know which tools only read data and which make changes, and can validate the results they get back. Write actions are now **confirmed via an elicitation prompt** before anything is saved (matching the in-app approval card), and several connection issues were fixed so the connector works reliably behind common reverse proxies and ingress setups.

## Reports

### Monthly Breakdown report, rounded out

The new Monthly Breakdown report picked up a lot of polish this release:

- **Income and expense categories are now grouped separately**, with a summary recap and **column sorting**.
- **Transfers** appear as their own rows, and the **Summary** and **Total** cells click straight through to the matching transactions — including a dedicated **uncategorized** drill-down.
- A toggle lets you **include or exclude the current (partial) month** from averages.
- The sticky category and name columns now **highlight cleanly on hover**, so it's easy to track a row across many months.

## Payees

### Apply a payee's default category to its past transactions

Setting a default category for a payee now offers to **backfill that category onto the payee's existing transactions** — choose none, only the uncategorized ones, or all of them. Transfers and split parents are never touched, so your manual work is safe, and Monize tells you how many transactions it changed. This is available when editing a payee, from the Auto-Assign Categories flow, and from the Auto-Merge Payees tool.

### Filter payees by what still needs categorizing

The Payees page has a new **category filter** (All Payees / No Default Category / Uncategorized Transactions) alongside the existing status filter, each with live counts, and surfaces an **uncategorized-transaction count** per payee so you can see at a glance where attention is needed.

## Net Worth

### See your asset-to-liability mix at a glance

Both the dashboard Net Worth chart and the Net Worth report gained a **"100% Stacked Bar" view** that shows the composition of your net worth as a percentage split of assets versus liabilities over time — so you can watch the ratio shift even when the absolute numbers are climbing. Your chosen view is remembered between sessions.

## Charts & Transactions

### Dismissible high/low markers and masked account numbers

Balance and forecast charts now flag their **highest and lowest points** with bubbles you can **dismiss with a click** to see the chart behind them. The Account Info widget now **masks the account number** by default, with an eye toggle to reveal it.

### Turn a split back into a regular transaction

Deleting one of the last two splits in the split editor now **converts the transaction back to a regular, single-category one**, after a clear confirmation, with the remaining split's category carried over. Works for both posted and scheduled transactions.

## Navigation & Consistency

### One consistent set of row actions everywhere

Per-row actions (edit, delete, merge, reconcile, post, close, and so on) were implemented differently on nearly every list. This release introduces **one shared row-actions system** adopted across Payees, Categories, Tags, Accounts, Scheduled Transactions, Currencies, Securities, Investment Transactions, and Institutions — with standardized icons, a consistent colour scheme, and **press-and-hold on touch screens** so phone-width rows no longer crowd or truncate the row's main content.

### Smoother settings and category editing

- The **mobile Settings navigation** swaps its horizontal scrolling tab strip for a compact **dropdown menu**.
- The **parent-category picker** is now a **searchable combobox**, matching the payee form, so it's quick to find a parent among many categories.

## Security & Sign-In

### Email verification for new registrations

When SMTP is configured, people who **sign up through the registration form** now receive a verification link and must confirm their address before logging in. The first user (the bootstrap admin) is auto-verified to avoid lockout, and admin-created users, invited delegates, and OIDC sign-ins are unaffected.

### More flexible OIDC email merging

A new **`OIDC_REQUIRE_VERIFIED_EMAIL`** setting (default `true`, so nothing changes unless you opt in) lets self-hosters whose identity provider doesn't assert `email_verified` — and who don't run SMTP — link an OIDC login to an existing local account by email. (Thanks **@WMP**.)

## Bug Fixes

- **Payee default categories** were silently wiped on an unchanged re-save; they now persist correctly, and transaction and payee forms send a proper "cleared" value instead of leaving stale data behind.
- **Charts in PDF exports** were rendering in a serif font instead of matching the web UI; the font is now pinned to sans-serif.
- Fixed **row-hover highlighting** on the sticky columns of the Monthly Breakdown report.
- Fixed the **brokerage close button, merge icon, and action-sheet colours**.

## Under the Hood

- **Security and dependency updates**, including bumping `hono` to clear a high-severity advisory and several grouped npm updates across the backend and frontend.
- **CI/CD hardening** — build actions moved to Node 24, container images now build in parallel, and several flaky end-to-end tests were stabilised.
- Expanded test coverage across the new i18n catalogs (parity suites pass for every locale), payee backfill, MCP schemas, and the AI approval flow.
