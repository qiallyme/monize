# v1.11.2

This release is all about the **AI Assistant and the MCP server**: a smaller, sturdier set of tools that behave identically across both surfaces, the ability to **bring your own AI subscription** to the in-app chat, and write actions — adding, editing, splitting, and deleting transactions and investments — that always ask before they save. It also brings **richer security data** (descriptions, your own tags, and geographic look-through), lets **transfers carry a category** so they show up properly in reports, and rounds things out with FX-rate fixes, layout fixes, and another pass of security hardening.

## AI Assistant & MCP

### Bring your own AI subscription to the in-app chat

The in-app AI chat can now be powered by **your own Claude (or other MCP-capable) subscription** through a new reverse MCP relay — no per-token API key required. Connect Claude CLI or Desktop to Monize's MCP server and ask your questions right inside the app. As you chat, Monize streams the same **progress chips** the native Assistant shows ("Looking up categories...", "Searching transactions..."), so you can see exactly which tools are running, and **write actions surface the same confirmation card** you approve before anything is saved. (Thanks **@WMP**.)

### A smaller, sturdier toolset that behaves the same everywhere

The financial tools shared by the AI Assistant and the MCP server were **consolidated and unified** so both surfaces expose exactly the same shapes:

- Account lookups collapsed into a single **`list_accounts`** (filter by type, open/closed, name, or id, with an assets/liabilities/net-worth summary).
- Transaction queries collapsed into a single **`list_transactions`** (search filters, grouping, transfers-only, names resolved internally, raw rows only when you ask for them).
- Transaction and investment writes collapsed into single **`manage_transactions`** and **`manage_investment_transactions`** tools (create/update/delete, 1–25 items per call), and the read tools were renamed with a consistent **`list_` prefix**.

Redundant and overlapping tools were removed, which leaves the model a clearer menu to choose from and makes it more likely to pick the right tool on the first try.

### Fewer dead ends, better first-try success

Several reliability improvements help the Assistant get it right the first time: the in-app Assistant gained five read tools it was missing (**list payees, holding details, generate report, list anomalies, and monthly comparison**), so it can answer "list my payees" or "any unusual spending this month?" directly instead of falling back to raw transaction listings. When an account or payee name doesn't resolve, Monize now appends **"did you mean" suggestions** so the model self-corrects, and the MCP server now states **today's date** in its instructions so relative ranges like "last month" resolve correctly on the first call.

### More it can do for you — always with approval

The Assistant (and connected MCP clients) can now **create investment transactions**, **create securities**, **split a transaction across categories**, and **create transactions in bulk** — each with the same human-in-the-loop confirmation as before. Bulk approval is now **gated behind a six-item threshold**: smaller batches confirm one at a time, larger ones in a single bulk card, so routine edits stay quick while big changes still get a clear preview.

## Securities & Portfolio

### Descriptions, your own tags, and exposure by tag

Securities can now carry a **description** (folded into the existing Lookup, so there's no separate fetch step) and your own **user tags**, and the portfolio can break your **exposure down by tag** — a flexible way to slice holdings however you think about them. (Thanks **@WMP**.)

### Geographic look-through for ETFs and funds

Because Yahoo and MSN don't publish country exposure for ETFs and funds, you (or the AI) can now **enter a country allocation by hand**. Portfolio reports roll these manual weightings up across all your holdings — splitting funds by look-through, attributing stocks by their listing exchange, and folding any shortfall into "Other" — to show a true **country (geographic) breakdown** of where your money is invested.

### Accurate cross-currency investment math

Cross-currency investment transactions now always use a **real foreign-exchange rate** instead of silently falling back to 1.0, and the rate lookup fetches a **bounded window** from Yahoo rather than the full history, so dated FX rates resolve quickly and correctly. Security currency is now read from the **live quote** rather than the exchange, fixing mislabelled currencies.

## Transfers & Reports

### Transfers can carry a category

A transfer can now **carry a category**, so it no longer disappears from the Monthly Breakdown and other category reports. The AI Assistant and MCP tools assign and persist that category correctly when creating or categorizing a transfer. (Thanks **@WMP**.)

## Interface

### Dropdowns and pages that fit the screen

- The **MultiSelect dropdown** now flips upward when there isn't room below and caps its height to the viewport, so options are never cut off.
- Fixed a long-standing issue where **pages overflowed the viewport** by the height of the header, causing an unnecessary scrollbar — including on the AI chat page.
- The AI chat input no longer **lags while typing** in long conversations.

### Quicker amount and split entry

- Currency inputs now carry an **in-field sign-toggle button**, so you can flip an amount between positive and negative without retyping it — available across the transaction, split, and scheduled-transaction forms.
- The **split control** in the scheduled-occurrence editor is now a **toggle** rather than a checkbox, matching the rest of the app, and the override editor's **amount label** was corrected.

## Security

### Continued hardening

- **Security headers** (including HSTS) are now applied to proxy responses generated by the middleware layer, closing a gap where some responses skipped them.
- The nightly **ZAP baseline scan** now runs against a production frontend build instead of the dev image, eliminating dev-only false positives, with documented suppressions for inherent ones; a **COEP header** and tightened ZAP rules were added.
- The shared **MCP daily write cap** is now enforced as a single per-user limit across all connection methods, so it can't be sidestepped.

## Bug Fixes

- **Security currency** was taken from the exchange rather than the live quote, mislabelling some holdings.
- Localized the remaining untranslated **investment action history** values and **investment account pair name** suffixes.
- Reduced **log noise** from a missing `MSN_API_KEY` at fetch time and added **ISO timestamps** to all frontend container logs.
- Fixed the **Security Performance chart** and several investment-transaction report issues.

## Under the Hood

- **CI/CD**: preview container images are now published for PRs and the main branch, fork-image builds were fixed, and a flaky transaction-deletion e2e test was stabilised with retry logic.
- **Security advisories**: forced patched `multer` 2.2.0 via overrides to clear seven high-severity advisories without downgrading NestJS.
- Added **branch-coverage tests** for import parsers, recurring-charge detection, and the new payee/security management tools, plus a **VAT/GST support design specification** for future work.
